## 1. 목표

P1.7로 확립된 ~900종목 Reference Universe 위에서 **자동 ontology relations**를 채운다.

- Schema 확장 (signal_direction / confidence / valid_period / metadata)
- relation_type 11 type 모두 활성 (peer / supply_*/group/theme/macro / **competitor / contract_supplier/customer / complementary / regulatory_link**)
- 5 source — sector match / DART 공시 / SEC 8-K / 뉴스 / 가격 상관관계
- LLM RAG로 본문에서 명시적 관계 추출 (DART 공시 contract = 신뢰도 0.95)
- 가족용 무과금 budget 안 (~$10/month)

P3 Stock Universe graph가 이 ontology를 시각화. P2 카드 RelationsSection이 ★ 발견 UX로 surface.

## 2. Locked Decisions

선행 spec §2의 8 원칙 + P1.6 specific:

1. **Phase 분리 (v0~v4)** — 한 번에 5 source 다 빌드 X. v0(schema) → v1(DART 공시) → v2(SEC 8-K) → v3(뉴스 + 상관관계) → v4(web search). 각 phase 별도 PR
2. **DART 공시 v1 우선** — 신뢰도 0.95 + 명시적 contract 정보 + 한국 공시 의무로 누락 적음. 가장 ROI 높은 시작점
3. **LLM 모델은 cheap-tier** — gpt-5-mini로 충분. relation extraction은 simple structured JSON output. 단 retry 한 번 (Pydantic schema fail 시)
4. **Universe 매칭 강제** — extracted ticker pair 둘 다 Reference Universe에 있어야 persist. Tier 3 ticker는 buffer table (`relation_candidates`)에 보류. 양쪽 다 universe 진입 시 auto-promote
5. **Manual curation UI 영구 deferred 유지** — 자동 path가 메인. confidence < 0.6은 review queue (read-only admin 노출만, P1.9+)
6. **양방향 onto_hook fix** — P1.5 sector_match는 신규 종목에서만 trigger (단방향 버그). v0에서 fix: universe-wide cross-match에서 양방향 row insert
7. **LangGraph X / RAG (pgvector) X** — plain Python async + 직접 ReAct loop + Pydantic schema. spec §4 + §9 그대로

## 6. 5 Source — Phase별 detail

### 6.1 v0 — sector_match auto cross-match

**의도**: P1.5 onto_hook은 신규 종목 등록 시 단방향 (from = 신규)만 등록. 이번 v0에서 양방향 + universe-wide.

```python
async def universe_wide_sector_match(db: AsyncSession) -> int:
    """
    Tier 1 + 2 universe 안에서 같은 (sector, industry_group) pair 모두
    peer row 양방향 등록. nightly idempotent (ON CONFLICT DO UPDATE).
    """
    universe = await db.execute(
        select(Stock).where(Stock.tier <= 2, Stock.is_delisted == False)
    )
    by_sector = group_by(universe.scalars(), key=lambda s: (s.sector, s.industry_group))
    
    inserted = 0
    for (sector, ig), stocks in by_sector.items():
        if not sector or sector == "Unknown":
            continue
        for a, b in combinations(stocks, 2):
            inserted += await _upsert_peer(db, a, b)  # 양방향
    return inserted
```

비용: 0 (DB query만). nightly 실행.

## 7. LLM RAG Pipeline 흐름

```
nightly cron (KR 02:00 KST + US 18:00 KST)
   │
   ├─ Step 1: Source Collection
   │     - DART 공시 (P1.7 KR universe × 최근 7일)
   │     - SEC 8-K filings (P1.7 US universe × 최근 7일)
   │     - 뉴스 (KR + US)
   │     - Web search (Tavily) — v4 옵트인
   │
   ├─ Step 2: Body chunk
   │     - 본문 ≥ 5K token이면 50% overlap 슬라이딩 chunk
   │     - Pydantic schema로 응답 강제
   │
   ├─ Step 3: LLM extraction (gpt-5-mini-tier)
   │     - Source별 prompt (DART / SEC / 뉴스 / web)
   │     - Pydantic ExtractionResult 모델 검증
   │     - JSON parse 실패 시 retry 1회
   │
   ├─ Step 4: Validation
   │     - from_ticker, to_ticker 정규화 (P1.5 normalize_ticker)
   │     - 양쪽 ticker 모두 Tier 1+2 universe 안에 있나? 
   │       - YES → stock_relations 직접 upsert
   │       - NO  → relation_candidates 테이블에 보류 (Tier 3 buffer)
   │     - confidence < 0.6 → review queue flag (admin only)
   │     - dedup: same (from, to, type, source_url) 중복은 첫 번째만
   │
   ├─ Step 5: Upsert
   │     - ON CONFLICT (from_stock_id, to_target, relation_type, source)
   │       DO UPDATE SET
   │         strength = (old.strength + new.strength) / 2,
   │         confidence = MAX(old.confidence, new.confidence),
   │         refreshed_at = NOW(),
   │         metadata = old.metadata || new.metadata
   │     - 양방향 등록 (signal_direction에 따라 부호)
   │
   ├─ Step 6: Correlation 검증 (옵션, v3+)
   │     - inverse signal 추출된 pair → 가격 corr 검증
   │     - corr 모순 시 confidence 차감 (0.7 → 0.5)
   │
   └─ Step 7: Tier 승격 (P1.7 hook)
         - relation_candidates에서 양쪽 universe 진입 pair 자동 promote
```

## 10. Edge cases & Invariants

| 케이스 | 처리 |
|---|---|
| LLM JSON 응답 깨짐 | Pydantic 검증 fail → retry 1회. 두 번째 fail 시 skip + log |
| Ticker 정규화 실패 | normalize_ticker `ValueError` → silent skip + log warning |
| 한쪽 ticker만 universe | relation_candidates 테이블에 보류. 양쪽 진입 시 promote |
| 같은 pair 중복 추출 (다른 source) | dedup by `(from, to, type, source_url)` 단 strength 평균 + confidence MAX |
| 공시 본문 fetch 실패 | log + 다음 ticker. nightly 재시도 |
| LLM hallucination — 실재 안 하는 ticker | universe 매칭에서 fail → silent drop |
| Inverse signal corr 검증 실패 | confidence 차감 (0.7 → 0.5), row는 유지 |
| 계약 만료 시점 데이터 없음 | `valid_until=null` + `is_active=true`. 1년 후 자동 inactive (cron) |
| 합병 / 분할 → ticker 변경 | 기존 relations 유지 + ticker mapping table (P1.7 합병 처리와 같이 진행) |
| Sector "Unknown" pair | sector_match는 skip. 단 LLM 추출 결과는 받음 (LLM이 sector 무관 contract 보면) |

### Adversarial (4 cases)

- LLM이 confidence > 1.0 또는 < 0 응답 → Pydantic ValidationError → retry
- 공시 본문 0 byte → skip
- corr 산출 시 가격 history 없음 → confidence 영향 X (corr 검증 skip)

### Integration (~3 cases, real dev DB)
- `ontology_backfill.py` 7일 backfill dry-run — relations ≥ 100 row
- candidate promote: Tier 3 ticker가 favorite 등록 → relations 즉시 만들어짐
- nightly cron 1회 실행 — 새 relations 추가됨 (or idempotent if 변경 없음)

### Smoke (opt-in, real LLM)
- DART 공시 1건 LLM extraction (~$0.01)
- SEC 8-K 1건 LLM extraction (~$0.005)
- 뉴스 1건 LLM extraction (~$0.001)

## 15. Risks

| 리스크 | 대응 |
|---|---|
| LLM hallucination — 실재 안 하는 contract 만듦 | universe 매칭 강제 + confidence < 0.6 review queue. 양방향 검증 (가격 corr 모순 시 차감) |
| DART 공시 본문 fetch quota / latency | dartlab 라이브러리 + 자체 throttle. 7일 backfill은 manual 1회 (cron 부담 X) |
| SEC EDGAR rate limit (10 req/sec) | 이미 P1.5 SecEdgarAdapter Semaphore(8) 적용. 추가 buffer 시 늘릴 수 있음 |
| LLM 비용 폭증 | gpt-5-mini-tier 사용 + 응답 size cap (max_tokens). 일 비용 모니터링 + 알림 |
| Relation duplication 폭발 (n choose 2) | sector_match는 same sector pair만. ~900 universe × sector quota → 산출 가능 |
| 한국어/영어 prompt mixing 시 LLM 혼란 | source별 prompt 분리 + 본문 언어와 일치 |
| 합병/분할 시 relation orphan | ticker mapping table + nightly cron이 cleanup |
