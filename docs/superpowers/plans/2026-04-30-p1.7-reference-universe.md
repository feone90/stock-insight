## 1. 목표

ontology가 emerge할 **시장 객관 universe (~900 종목)** 을 자동 seed + nightly refresh.

- KOSPI 200 + KOSDAQ 200 + S&P 500 시총/index 기반 + GICS sector quota
- 외부 API/LLM 비용 0 (P1.5 어댑터 활용)
- 사용자 검색/카드 view 시 Tier 3 → Tier 2 자동 승격
- 상장폐지/합병 graceful 처리

## 2. Locked Decisions

선행 spec §2의 8 원칙이 본 plan의 baseline. P1.7 specific 결정 4개:

1. **3-Tier 모델** — Tier 1 reference core (~900) / Tier 2 user-touched (자동 승격) / Tier 3 latent (메타 only, ontology cross-match X)
2. **Sector quota** — GICS 11 sector마다 minimum 30종목 보장. 단순 시총 top은 IT/Financial 편향
3. **Source 자동화** — KR=`dartlab.listing()`, US=wikipedia S&P 500 list. 비용 0
4. **Refresh cycle** — KR 시총 변동: nightly. US S&P 500 membership: 분기 (S&P committee 변경 주기)

## 5. Schema 변경

`Stock` 모델에 추가 (alembic migration):

```sql
ALTER TABLE stocks
  ADD COLUMN tier SMALLINT NOT NULL DEFAULT 3,
  -- 1 = reference core (KOSPI 200 / KOSDAQ 200 / S&P 500 + sector quota)
  -- 2 = user-touched (favorite 또는 카드 view)
  -- 3 = latent (메타만, ontology cross-match X)

  ADD COLUMN industry_group VARCHAR(100),
  -- GICS sub-industry. sector + industry_group 함께 쓰면 ontology 정밀도 ↑

  ADD COLUMN market_cap NUMERIC(18, 2),
  -- KRW 또는 USD 시가총액. universe selection + nightly 변동 반영용

  ADD COLUMN avg_volume_30d NUMERIC(18, 2),
  -- 일평균 거래량 30일. liquidity 컷오프

  ADD COLUMN is_delisted BOOLEAN NOT NULL DEFAULT FALSE,
  -- 상장폐지 / 거래정지. row 유지하되 카드 진입 차단

  ADD COLUMN tier_updated_at TIMESTAMP,
  -- Tier 변경 audit (자동 승격 추적용)

  ADD COLUMN universe_source VARCHAR(50);
  -- "kospi200_marketcap" / "kosdaq200_marketcap" / "sp500_index" /
  -- "sector_quota_kr" / "sector_quota_us" / "user_promoted" / "manual"

CREATE INDEX ix_stocks_tier ON stocks(tier);
CREATE INDEX ix_stocks_sector_tier ON stocks(sector, tier);
```

```python
def select_kr_universe(min_per_sector: int = 30) -> list[StockMeta]:
    listing = dartlab.listing().to_pandas()

    # Tier 1 candidates per market: top 200 by market_cap
    kospi = listing[listing.market == "KOSPI"].nlargest(200, "market_cap")
    kosdaq = listing[listing.market == "KOSDAQ"].nlargest(200, "market_cap")
    candidates = pd.concat([kospi, kosdaq])

    # Apply GICS sector quota
    selected = apply_sector_quota(candidates, listing, min_per_sector)
    return selected
```

### 6.3 Sector quota algorithm

`backend/app/services/universe/sector_quota.py`:

```python
def apply_sector_quota(
    candidates: pd.DataFrame,
    universe_pool: pd.DataFrame,
    min_per_sector: int,
) -> pd.DataFrame:
    """
    Ensure every GICS sector has >= min_per_sector stocks in candidates.
    If a sector is under-represented, top up from the universe_pool.
    """
    by_sector_in_candidates = candidates.groupby("sector").size()
    by_sector_in_pool = universe_pool.groupby("sector")

    additions = []
    for sector, group in by_sector_in_pool:
        current = by_sector_in_candidates.get(sector, 0)
        deficit = min_per_sector - current
        if deficit > 0:
            extra = (
                group[~group.ticker.isin(candidates.ticker)]
                .nlargest(deficit, "market_cap")
            )
            additions.append(extra)
    if additions:
        candidates = pd.concat([candidates, *additions])
    return candidates
```

### 6.4 Liquidity 컷오프

```python
LIQUIDITY_CUT_KRW = 100_000_000  # 일평균 거래대금 1억원 미만 제외
```

## 7. Universe Seed — US

### 7.1 Source — S&P 500 list

wikipedia `https://en.wikipedia.org/wiki/List_of_S%26P_500_companies` HTML table에서 ticker + sector.

```python
import httpx
from bs4 import BeautifulSoup

URL = "https://en.wikipedia.org/wiki/List_of_S%26P_500_companies"

async def fetch_sp500_list() -> list[StockMeta]:
    async with httpx.AsyncClient() as client:
        r = await client.get(URL, headers={"User-Agent": "StockInsight"})
    soup = BeautifulSoup(r.text, "html.parser")
    table = soup.find("table", {"id": "constituents"})
    rows = []
    for tr in table.tbody.find_all("tr")[1:]:
        cells = tr.find_all("td")
        if len(cells) >= 4:
            ticker = cells[0].text.strip()
            name = cells[1].text.strip()
            gics_sector = cells[2].text.strip()
            gics_sub = cells[3].text.strip()
            rows.append({
                "ticker": ticker, "name": name,
                "market": "US",
                "sector": _normalize_gics(gics_sector),
                "industry_group": gics_sub,
            })
    return rows
```

```python
async def promote_to_tier_2(ticker: str, db: AsyncSession) -> None:
    """
    User가 favorite 등록 또는 카드 view 시 호출.
    Tier 3 → Tier 2 승격 (이미 Tier 1/2이면 noop).
    """
    stmt = (
        update(Stock)
        .where(Stock.ticker == ticker, Stock.tier == 3)
        .values(
            tier=2,
            tier_updated_at=func.now(),
            universe_source="user_promoted",
        )
    )
    await db.execute(stmt)
    await db.commit()
```

## 9. Nightly Refresh

`backend/app/services/universe/refresh.py`:

```python
async def nightly_universe_refresh() -> dict:
    """
    매일 새벽 (KR 시장 open 전) — 시총 변동 반영.
    Tier 1 진입/탈락 처리.
    """
    kr_universe = select_kr_universe()
    us_universe = select_us_universe()  # quarterly이므로 cache hit

    async with async_session() as db:
        # 1. 신규 진입 — DB에 없거나 tier > 1
        for s in kr_universe + us_universe:
            await _upsert_stock_tier1(db, s)

        # 2. 탈락 — tier 1인데 universe에 없음 → tier 2 강등 (user_promoted 보존)
        await _demote_dropouts(db, kr_universe + us_universe)

        await db.commit()
    return {"promoted": ..., "demoted": ...}
```

## 11. Edge cases & Invariants

| 합병 / 분할 | 메타 update + ontology relations migration script (P1.6 phase F) |
| 시총 급변 (top 200 진입/탈락) | nightly reflect |
| 사용자 검색 Tier 3 → 2 | `_get_or_register_stock` 호출 시 즉시 promote |
| 사용자 favorite 제거 | tier 유지 (한 번 view한 종목은 ontology에 남김) |
| dartlab listing fetch 실패 | 기존 universe 유지 + log warning. nightly retry |
| wikipedia fetch 실패 | 기존 S&P 500 cache 유지 |
| 중복 ticker (KR/US 동일 ticker) | `(ticker, market)` UNIQUE — 이미 P1 schema |
| sector="Unknown" (SIC 매핑 미스) | universe 편입은 가능, sector quota 계산에서 별도 bucket |

## 15. Out of Scope (P1.7 안에서 안 하는 것)

- Relation extraction — P1.6에서 (universe seed 끝나야 cross-match 가능)
- KOSPI 200 정적 list 활용 (시총 top 자동 계산이 더 robust)
- Tier 2/3 universe 확장 자동화 (user-promoted 외 자동 진입은 P1.6+에서 ontology 1-hop neighbor 자동 편입 시 진행)

- ETF / sector index 노드 (universe는 stock only) — v2.1+
